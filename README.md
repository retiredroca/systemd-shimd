# systemd-free systemd | systemd-shimd

A lightweight systemd compatibility layer for systemd-free Linux distributions.
Translates systemd commands and D-Bus calls into equivalent operations for the
host init system (OpenRC, Runit, s6, sysvinit).

Zero abstraction overhead — all backend logic uses C++ template dispatch
(`template<Init> struct Backend`) with a direct `switch(detect())` runtime
dispatcher. No vtables, no opaque types, no resolver indirection.

## Components

| Component | Description |
|-----------|-------------|
| `systemctl` | 33+ subcommands: start, stop, restart, status, enable, disable, mask, unmask, reload, try-restart, reenable, is-active, is-enabled, is-failed, is-system-running, kill, cat, show, edit, list-units, list-unit-files, list-dependencies, add-wants, preset, link, daemon-reload, daemon-reexec, reset-failed + get-default, set-default, reboot, poweroff, halt |
| `journalctl` | view logs with -f, -n, -u, --since, --until |
| `systemd-shimd` | D-Bus daemon implementing org.freedesktop.systemd1.Manager and org.freedesktop.systemd1.Unit |
| `libsystemd.so.0` | LD_PRELOAD shim: sd_notify, sd_journal, sd_daemon, sd_id128, sd_booted |
| `systemd-analyze` | boot time and service timeline analysis |

## Source tree

```
src/
  systemd-shimd.hpp  # single header: template<Init> struct Backend<>
                     # + common helpers, systemd parser, init script generator
  systemctl.cpp      # CLI main
  journalctl.cpp     # CLI main + log reader
  dbus-service.cpp   # D-Bus daemon
  libsystemd.cpp     # LD_PRELOAD shim (extern "C")

```

## Build

```
cmake -B build -DCMAKE_BUILD_TYPE=Release
cmake --build build -j$(nproc)
```

Dependencies: `cmake`, `g++` (C++20), `libelogind` (for D-Bus daemon).

Install:

```
sudo cmake --install build
```

systemctl reenable sshd             # disable then enable

systemctl unmask sshd
systemctl is-active sshd
systemctl is-enabled sshd
systemctl is-failed sshd
systemctl is-system-running

systemctl cat sshd

systemctl list-units
systemctl list-unit-files
systemctl list-dependencies sshd
systemctl add-wants multi-user.target sshd  # advisory wants dependency
systemctl preset sshd
systemctl link /path/to/custom.service

systemctl daemon-reload
systemctl daemon-reexec

systemd-analyze

## Auto-generated init scripts

When a package installs only a systemd `.service` file (no corresponding init
script), systemd-shimd auto-generates an equivalent script on first access.
Supported for all backends:

| Init | Output path | Format |
|---|---|---|
| OpenRC / sysvinit | `/etc/init.d/<name>` | `openrc-run` script with `command`/`supervisor` |
| Runit | `/etc/sv/<name>/run` + `finish` | `#!/bin/sh -e\nexec <cmd> <args> 2>&1` |
| s6 | `/etc/s6-rc/<name>/run` | `#!/bin/sh -e\nexec <cmd> <args>` |

Parsed from the systemd unit:

| systemd key | OpenRC mapping |
|---|---|
| `[Unit] Description` | `description=""` |
| `[Service] ExecStart` | `command=""` + `command_args=""` |
| `[Service] ExecStop` | `stop()` override with `start-stop-daemon --stop` |
| `[Service] ExecReload` | `reload()` override with `--signal HUP` |
| `[Service] Type=simple` | `supervisor="supervise-daemon"` |
| `[Service] Type=forking` | `command_background=true` + `pidfile=""` |
| `[Service] Type=oneshot` | direct `command` without supervisor |
| `[Service] User` / `Group` | `command_user=""` |
| `[Service] WorkingDirectory` | `directory=""` |
| `[Service] Environment` | `export VAR=val` |
| `[Service] PIDFile` | `pidfile=""` |

Generated scripts are marked `# Auto-generated by systemd-shimd` and cleaned
up on `systemctl disable`. Never overwrites an existing init script.

## LD_PRELOAD

```
SYSTEMD_SHIMD_DEBUG=1 LD_PRELOAD=/usr/lib/libsystemd.so.0 your-program
```

Provides sd_notify, sd_journal_print, sd_listen_fds, sd_is_socket, sd_id128,
and sd_booted for programs expecting systemd's libsystemd at runtime.

sd-bus, sd-event, and sd-login symbols are not included — those are provided
by elogind.
